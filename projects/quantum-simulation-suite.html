Executive Summary

This project represents a unified, web-based platform designed to demonstrate and execute quantum algorithms for complex physical simulations. Built on the Trame framework, it integrates two distinct scientific domains—Electromagnetics and Fluid Dynamics—into a single, user-friendly interface.

The application empowers users to configure simulation parameters, generate quantum circuits, execute them on real quantum hardware or high-performance simulators, and visualize the results in real-time 3D. It serves as a bridge between classical engineering simulations and the emerging capabilities of quantum computing.

High-level architecture

The centralized dashboard connecting distinct physics modules (EM & QLBM) via a unified Trame interface.

Core Modules

1) Electromagnetic (EM) Scattering App

  • Function: Simulates electromagnetic wave propagation and scattering phenomena.
  • Methodology: Utilizes quantum estimation primitives (EstimatorV2) to calculate expectation values for field components (e.g., $E_z$, $H_x$) over time.
  • Features: Supports custom impulse/Gaussian initial states, time-stepping visualization, and integration with quantum hardware for retrieving physical observables.

EM scattering workflow

From geometry definition to circuit execution and field visualization.

2) Quantum Lattice Boltzmann Method (QLBM) App

  • Function: Performs 3D fluid dynamics simulations using the Lattice Boltzmann Method.
  • Methodology: Leverages a hybrid quantum-classical computing architecture to solve fluid flow equations.
  • Features: Capable of running on CPU/GPU backends for demonstration or scaling up to quantum-accelerated solvers for complex fluid interactions.

Deep Dive: QLBM Technical Architecture

The QLBM module is a pioneering solver reimagining Computational Fluid Dynamics (CFD) for the quantum era.

Hybrid Quantum–Classical Architecture

The application leverages the specific strengths of different processors to solve the Boltzmann transport equation:

  • Quantum collision operator: the most computationally intensive segment—the collision step—is offloaded to quantum backends. The app maps particle distribution functions to quantum states, using interference to perform complex collision dynamics.
  • Classical streaming: propagation and boundary condition enforcement are handled efficiently by classical CPU/GPU routines.

Backend Flexibility

  • NVIDIA CUDA-Q: GPU-accelerated statevector simulation.
  • IBM Quantum & IonQ: gate-based and trapped-ion hardware execution.
  • Qiskit Aer: local, high-performance simulation for development.

Key Features & Technical Highlights

Hardware-agnostic quantum execution

  • Superconducting qubits: via IBM Qiskit Runtime (Estimator primitive).
  • Trapped-ion qubits: via IonQ Provider (e.g., Aria, Forte).
  • GPU acceleration: via NVIDIA CUDA-Q.

Real-time feedback & asynchronous processing

To address the latency of quantum jobs, the application uses a non-blocking architecture. Heavy computational tasks are offloaded to background threads, while a polling mechanism monitors job status (Queued, Validating, Running). This keeps the UI responsive while tracking progress from transpilation to result retrieval.

Scientific accuracy & error mitigation

  • Error mitigation: Zero Noise Extrapolation (ZNE) and Dynamical Decoupling (DD).
  • Validation: quantum results benchmarked against classical FDTD reference solutions.

Interactive “Digital Twin” visualization

  • 3D volumetric rendering: slice, zoom, and rotate fluid density and velocity fields directly in the browser.
  • Time-series analysis: dynamic charts track the evolution of quantum observables step-by-step.

Technology Stack

Quantum SDKs

  • Qiskit SDK: circuit construction and transpilation.
  • CUDA-Q (NVIDIA): GPU-accelerated hybrid simulations.
  • Qiskit Runtime (IBM): primitive execution with error mitigation.
  • IonQ Provider: job submission to trapped-ion computers.
  • Qiskit Aer: local statevector simulation.
  • ADAPT-AQC: adaptive approximate quantum compiling for circuit optimization.

Application infrastructure

  • Frontend/UI: Trame (Vuetify 3), PyVista, Plotly.
  • Backend: Python.
  • Deployment: Docker containers, Nginx (reverse proxy), cloud-native support (Hugging Face Spaces).

EM App Workflow: From Maxwell’s Equations to Quantum Simulation

The EM App simulates electromagnetic wave propagation by mapping Maxwell’s equations onto a qubit lattice. The workflow transitions from physical geometry definition to execution of quantum circuits through four stages.

1) Problem setup & geometry definition

  • Grid configuration: the domain is discretized (e.g., $4\times4$ or $8\times8$). The app computes required qubits $n_q = \lceil \log_2(N_x) \rceil$ for spatial coordinates.
  • Initial excitation: impulse (delta-source at $(x,y)$) or Gaussian wavepacket with width $\sigma$.
  • Simulation parameters: total time $T$, time step $dt$, and target component ($E_z$, $H_x$, $H_y$).

2) Quantum circuit generation

  • State preparation: encode the initial field distribution into $|\psi(0)\rangle$ via state preparation routines.
  • Time evolution: evolve via $U(t)=e^{-iHt}$ using Hamiltonian simulation/Trotterization.
  • Circuit optimization: optional ADAPT-AQC optimization to reduce depth while preserving fidelity.

3) Execution & measurement

  • Backend selection: Qiskit Aer for local validation, or IBM Quantum / IonQ for hardware execution.
  • Measurement strategy: magnitude extraction + sign recovery via interference/Hadamard-test style circuits.
  • Error mitigation: ZNE and dynamical decoupling on hardware runs.

4) Visualization & analysis

  • Time-series analysis: track a monitor point over time for expensive hardware runs.
  • 3D field visualization: full-field rendering in simulator mode using PyVista.
  • Export: results exportable (e.g., CSV) and cacheable for comparison.

QLBM App Workflow: From Definition to Digital Twin

The QLBM App guides users through a structured scientific workflow designed to make complex quantum fluid dynamics accessible. The process is divided into five stages.

1) Problem definition & geometry setup

  • Geometry selection: predefined domains (cube, rectangular domain with heated box, etc.).
  • Boundary conditions: periodic, bounce-back walls, etc.
  • Grid resolution: choose $32^3$, $64^3$…; the app shows qubit requirements for feasibility.

2) Initial state configuration

  • Distribution types: Gaussian, sinusoidal, multi-delta initializations.
  • Parameter tuning: interactive controls for width $\sigma$ and centers $(c_x,c_y,c_z)$ with real-time preview.

3) Advection field (velocity) setup

  • Velocity presets: uniform, shear, swirl/TGV.
  • Custom expressions: input expressions like $0.5\cdot\sin(2\pi x)$.

4) Backend selection & execution

  • CPU demo: instant approximation on CPU-only environments.
  • CUDA-Q: GPU-accelerated statevector simulation.
  • IBM Quantum / IonQ: hardware execution with job submission and status polling.

5) Visualization & analysis (the “Digital Twin”)

  • Volumetric rendering: interactive 3D density view.
  • Time evolution: playback controls animate the flow.
  • Inspection: isosurface thresholding and point probing.
  • Export: save VTK for ParaView or MP4 animations.
